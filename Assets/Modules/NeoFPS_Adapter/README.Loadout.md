# NeoFPS_PlayerLoadoutAdapter

## Назначение

Проектный adapter для точной настройки стартового оружия и боеприпасов
через штатный `IInventory.ApplyLoadout(...)` без правок NeoFPS.

## Как подключить

1. Добавьте `NeoFPS_PlayerLoadoutAdapter` на корневой объект игрока с NeoFPS inventory.
2. Заполните `startupItemPrefabs` только теми prefab-предметами, которые должны быть у игрока на старте.
3. Для жёсткой замены vendor-loadout оставьте `replaceExistingStartingItems = true`.

## Как работает интеграция

- Adapter срабатывает в `Awake`.
- Он находит `IInventory` на объекте игрока.
- Дальше вызывает `ApplyLoadout(prefabs, replace: true)` до `Start()` инвентаря NeoFPS.
- Это важно, потому что `FpsInventoryBase` финально собирает стартовый inventory в `Start()`,
  где добавляет `m_StartingItems` и `m_AdditionalPrefabs`.
- Вызов в `Awake` позволяет подменить набор ещё до финальной инициализации стартового inventory,
  не трогая код NeoFPS и не создавая лишние предметы с последующей зачисткой.
- За счёт этого штатные `m_StartingItems` NeoFPS не попадают в реальный инвентарь,
  а игрок стартует с проектным loadout.

## Однократное применение

- Adapter хранит внутренний флаг `applied`.
- Если по какой-то причине `Awake` будет вызван повторно на том же экземпляре персонажа,
  `ApplyLoadout` второй раз не выполнится.
- При рестарте уровня создаётся уже новый экземпляр player prefab, и loadout применяется заново
  ровно один раз для нового жизненного цикла персонажа.

## Заметка по расширяемости

- На текущем этапе loadout не зашит в коде: он уже вынесен в сериализуемый массив
  `startupItemPrefabs`, который настраивается в Inspector.
- Для текущей задачи этого достаточно: поведение простое, а менять pistol/ammo можно без
  переписывания adapter.
- Следующий аккуратный шаг, когда появятся разные режимы или уровни: вынести этот массив
  в отдельный `ScriptableObject` с набором item prefab ссылок и подставлять его в adapter.
- Отдельную систему `item + quantity` сейчас добавлять рано, потому что количество патронов
  уже штатно инкапсулировано внутри самих NeoFPS inventory item prefabs.
